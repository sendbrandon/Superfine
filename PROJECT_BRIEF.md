# SUPERFINE — Project Brief

## What this is

A single-page web artifact for a Met Gala 2026 counter-event called **The Guest List**. The whole site is one alphabetical scrolling list of Black dandies through history (Frederick Douglass → Lewis Hamilton), set in italic Times New Roman on ivory parchment. One field at the top: **add yourself, $1**. Your name appears in the list, alphabetized, permanently.

It's Drop #002 in a portfolio of MSCHF-style cultural artifacts. Drop #001 is Sunday-OS (already shipped — separate repo).

## Cultural argument

The Met Gala 2026 theme is **"Superfine: Tailoring Black Style"** (curated by Monica L. Miller, drawing on her book *Slaves to Fashion: Black Dandyism and the Styling of Black Diasporic Identity*).

The actual Met Gala has 400 invitees. Anna Wintour gatekeeps the list. THE GUEST LIST is the counter-institution: **anyone with $1 can claim a permanent place in the lineage of Black dandyism**, alphabetically sorted next to Frederick Douglass, Cab Calloway, James Baldwin, Pharrell.

The closer: *"the only invitation list more exclusive than the one Anna keeps — this one already includes the dead."*

This is on purpose transgressive. We are commodifying participation in Black dandyism's lineage. We are putting the dead on a paid list. We are mocking the institution while honoring the subject. **Don't soften this. The transgression IS the artifact.**

## Functional spec

### The list

- Site renders one continuous alphabetical list of names
- ~150 names hand-curated as the seed (in `lib/curated-names.ts`)
- User-paid additions merge into the alphabetical scroll, persistent
- Articles ("the", "a") stripped for sort key; names with non-ASCII characters (é, ñ, ï) sort correctly via `localeCompare`

### The three tiers

Click any tier tile, fill in the name, pay via Stripe Checkout:

| Tier | Price | What you get |
|------|-------|--------------|
| Take a seat | $1 | Name in the alphabetical scroll |
| The Ribbon | $25 | Name in the scroll + oxblood `▎` glyph next to it |
| Patron | $100 | Name in a "Patrons" block at the top of the list, with optional dedication line |

### The viral mechanics

After payment succeeds, three things fire:

1. **`/api/confirm`** verifies the Stripe session was actually paid (so the URL can't be spoofed), then `RPUSH`-es the entry to KV under key `superfine:guest-list`
2. User is redirected to `/?added={name}&tier={tier}`
3. Page auto-opens a **TicketReveal modal** showing a personalized 1080×1920 ticket image (the user's name framed in oxblood rules, with the names directly above and below them in the list as their "lineage") with Save / Twitter / Facebook / Web Share / Copy buttons

The personalized ticket image is generated by `/api/ticket` via `@vercel/og` — a dynamic PNG route that fetches the user's neighbors from KV at request time.

A **live counter** under the wordmark polls `/api/count` every 30s and shows the total names on the list. Social proof.

### Moderation

`lib/moderate.ts` rejects the obvious offenders BEFORE any Stripe session is created: unambiguous slurs, URLs, HTML, repeated-character spam, names under 2 letters. Open enrollment + $1 entry guarantees someone tries something gross — the filter handles ~90%.

### The "no donation" decision

We considered committing 50% of revenue to a Black design org (Schomburg / Harlem Fashion Row). Brandon explicitly chose **no donation**. The artifact is raw and transgressive on purpose. Don't add a donation flow.

## Tech stack

- Next.js 14 (App Router)
- React 18, TypeScript
- Plain CSS (NO Tailwind — see Vibe Brief for why)
- Stripe Checkout for payments
- `@vercel/kv` client (works against Upstash Redis, the new backend)
- `@vercel/og` for dynamic ticket image rendering
- Tinos font bundled locally for true-Times rendering in Edge runtime

## Required env vars

```
STRIPE_SECRET_KEY=sk_test_…             # or sk_live_… in production
NEXT_PUBLIC_BASE_URL=https://superfine.vercel.app
KV_REST_API_URL=…                       # auto-injected by Upstash Vercel integration
KV_REST_API_TOKEN=…                     # auto-injected
```

Without these, the action gracefully degrades — clicking Add routes through a mock-success path with in-memory persistence (vanishes per serverless invocation). Useful for local dev. Production needs all four.
